Banking & Fintech
Citadel Plan
StellarBank Pvt. Ltd.
📍 Mumbai, Maharashtra
// The Challenge
A nation-state APT group (tracked as TUNGSTEN-9) compromised a third-party payroll vendor and used their legitimate VPN credentials to move laterally into StellarBank's internal payment systems. Traditional SIEM tools produced no alerts — the attacker blended in as a legitimate contractor session.
// Ironwood's Agentic Response
Ironwood's UEBA agent flagged a 340% deviation in the contractor account's access patterns at 02:14 IST. The Identity Protection agent cross-correlated this with an unusual SWIFT message drafting activity. Within 0.8 seconds, the Isolation Agent quarantined 47 affected endpoints, revoked all active sessions, and triggered an automated CERT-In incident report. The Threat Intel agent attributed the attack to TUNGSTEN-9 within 4 minutes using dark web IOC correlation.

All 14 production lines continued without a second of interruption. The SOAR playbook automatically notified plant management and legal within 90 seconds.

Healthcare & Pharma
Fortress Plan
MediBridge Hospitals
📍 Bengaluru, Karnataka
// The Challenge
An insider threat — a disgruntled database administrator — was exfiltrating patient records (2.4M records containing Aadhaar-linked health data) to a private cloud storage account. The exfiltration was designed to mimic routine database backup traffic to evade detection. DPDP Act exposure was estimated at ₹12Cr in penalties.
// Ironwood's Agentic Response
Ironwood's DLP agent detected a 2,300% spike in outbound encrypted traffic from the DBA's workstation at 23:40 IST. The UEBA agent simultaneously identified that the destination IP was not in the approved backup destinations registry. The agent automatically revoked the user's database credentials, blocked the upload mid-transfer (only 0.3% of records were transferred before blocking), and escalated to HR and Legal with full forensic evidence packages.

// Ironwood's Agentic Response
Ironwood's API Security agent detected the anomalous login velocity at T+3 seconds and applied progressive friction — first CAPTCHA, then SMS OTP, then full block — to suspected bot traffic while leaving genuine users unaffected. The WAF agent dynamically updated blocking rules every 90 seconds as the attackers rotated IP ranges. Of 280,000 malicious attempts per minute, 99.98% were blocked. Zero legitimate customers were locked out during the entire 6-hour attack window.

// Agentic Response Timeline

From breach to containment — how it happens

Ironwood's multi-agent architecture runs all response stages in parallel, not sequentially. Here's the anatomy of a typical incident response.

🔍
T + 0.0s — Detection
Anomaly Detected by Behavioural AI
The UEBA agent detects a statistically significant deviation from the baseline. Simultaneously, EDR, NDR, and DNS agents correlate signals. A threat confidence score is computed within 200ms.
⚡
T + 0.8s — Containment
Autonomous Isolation & Credential Revocation
If confidence score exceeds threshold, the Isolation Agent quarantines affected endpoints, the IAM Agent revokes active sessions, and firewall rules are updated — all within one second, all without human approval.
🧠
T + 4 minutes — Investigation
Automated Root Cause Analysis
The Forensics Agent reconstructs the full attack timeline, correlates with threat intelligence databases, and attributes the attack vector. A preliminary incident report is generated in plain language.
📋
T + 90 seconds to 6 hours — Notification & Remediation
CERT-In Filing, Stakeholder Alerts, System Recovery
SOAR playbooks handle CERT-In mandatory reporting (within required timelines), notify legal, HR, and executives, and guide IT teams through verified remediation steps. Clean restore points are validated before systems come back online.
